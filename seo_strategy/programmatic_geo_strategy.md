# Стратегия построения SEO-структуры Bablocar

## 1. Структура сайта

Сначала фиксируем дерево структуры. Пояснения к уровням идут ниже.

```text
Уровень 1
Главная
/
│
└── Уровень 2
    Федеральные хабы интентов
    │
    ├── /zaim-pod-pts/
    │   Займ под ПТС
    │   │
    │   ├── Федеральные модификаторы интента
    │   │   ├── /zaim-pod-pts/bez-spravok/
    │   │   ├── /zaim-pod-pts/s-plohoi-ki/
    │   │   └── /zaim-pod-pts/bez-stoyanki/
    │   │
    │   └── Уровень 3
    │       Гео-страницы этого интента
    │       │
    │       ├── /samara/zaim-pod-pts/
    │       │   Займ под ПТС в Самаре
    │       │   │
    │       │   └── Уровень 4
    │       │       Гео-модификаторы
    │       │       ├── /samara/zaim-pod-pts/bez-spravok/
    │       │       ├── /samara/zaim-pod-pts/s-plohoi-ki/
    │       │       └── /samara/zaim-pod-pts/bez-stoyanki/
    │       │
    │       ├── /kazan/zaim-pod-pts/
    │       │   Займ под ПТС в Казани
    │       │
    │       └── /moskva/zaim-pod-pts/
    │           Займ под ПТС в Москве
    │
    ├── /kredit-pod-pts/
    │   Кредит под ПТС
    │   │
    │   └── Гео-страницы этого интента
    │       ├── /samara/kredit-pod-pts/
    │       ├── /kazan/kredit-pod-pts/
    │       └── /moskva/kredit-pod-pts/
    │
    └── /dengi-pod-pts/
        Деньги под ПТС
        │
        └── Гео-страницы этого интента
            ├── /samara/dengi-pod-pts/
            ├── /kazan/dengi-pod-pts/
            └── /moskva/dengi-pod-pts/
```

Вспомогательные связи не смешиваем с главным деревом. Они относятся к навигации и будущему составу страниц:

```text
/samara/
│
├── /samara/zaim-pod-pts/
├── /samara/kredit-pod-pts/
├── /samara/dengi-pod-pts/
├── /samara/zaim-pod-avto/
└── /samara/avtolombard/
```

```text
/blog/
│
├── /blog/kak-vzyat-zaym-pod-pts/
├── /blog/dokumenty-dlya-zayma-pod-pts/
├── /blog/chto-budet-esli-ne-platit/
└── /blog/kak-proverit-avto-na-zalog/
```

Статьи ссылаются на федеральные хабы и, где уместно, на выбор города.

## 2. Пояснение к структуре

### Федеральные хабы

Федеральный хаб - это родительская страница коммерческого интента.

Пример:

```text
/zaim-pod-pts/
```

Это полноценный лендинг + хаб.

Он нужен, чтобы:

- закрыть общий запрос без города;
- объяснить продукт;
- покрыть алиасы интента;
- дать ссылки на города;
- дать ссылки на соседние интенты;
- дать ссылки на статьи и FAQ;
- быть структурным центром кластера.

Состав страницы федерального хаба обсуждается отдельно от дерева. Базовый набросок:

```text
/zaim-pod-pts/
│
├── первый экран и форма заявки
├── условия
├── документы
├── как работает заявка
├── преимущества
├── FAQ
├── связанные интенты
│   ├── /kredit-pod-pts/
│   ├── /dengi-pod-pts/
│   └── /zaim-pod-avto/
│
├── популярные города
│   ├── /samara/zaim-pod-pts/
│   ├── /kazan/zaim-pod-pts/
│   └── /moskva/zaim-pod-pts/
│
└── статьи по теме
```

### Гео-страницы

Гео-страницы - основная масса programmatic SEO.

Пример:

```text
/samara/zaim-pod-pts/
```

Задачи страницы город + интент:

- закрыть гео-коммерческий запрос;
- дать локальную релевантность;
- вести в заявку;
- ссылаться на федеральный хаб;
- ссылаться на страницу города;
- ссылаться на соседние интенты в этом городе;
- ссылаться на разрешенные модификаторы.

Пример связей:

```text
/samara/zaim-pod-pts/
│
├── /zaim-pod-pts/
├── /samara/
├── /samara/kredit-pod-pts/
├── /samara/dengi-pod-pts/
└── /samara/zaim-pod-pts/bez-spravok/
```

### Страницы с модификаторами

Модификаторы применяются к интентам по правилам.

Примеры:

```text
/zaim-pod-pts/bez-spravok/
/samara/zaim-pod-pts/bez-spravok/
```

Отдельные страницы с модификаторами создаются только если связка проходит фильтр применимости.

## 3. Главный принцип

Мы строим структуру не под каждый отдельный ключ, а через систему:

```text
очищенная семантика
-> коммерческие интенты
-> алиасы интентов
-> модификаторы
-> правила применимости
-> федеральные хабы
-> гео-страницы
-> страницы с модификаторами
-> статьи / FAQ
```

Основная модель для programmatic SEO:

```text
города x коммерческие интенты x разрешенные модификаторы
```

## 4. Коммерческий интент

Коммерческий интент - это связка:

```text
тип продукта + объект залога
```

Типы продукта:

- займ;
- кредит;
- деньги;
- автоломбард;
- заложить.

Важное правило:

```text
займ != кредит != деньги
```

Это разные коммерческие интенты, а не алиасы друг друга.

Примеры:

- займ под ПТС;
- кредит под ПТС;
- деньги под ПТС;
- займ под авто;
- кредит под авто;
- деньги под авто;
- займ под машину;
- кредит под машину;
- деньги под машину.

## 5. Объекты залога

Объект залога - это то, под что пользователь хочет получить деньги.

Базовые объекты:

- ПТС;
- авто;
- машина;
- грузовик;
- спецтехника;
- мотоцикл;
- коммерческий транспорт.

Правила объединения:

| Формулировка | Объект |
| --- | --- |
| ПТС | ПТС |
| авто / автомобиль / автомобиля | авто |
| машина / машины | машина |
| машину | не объединяем с авто |
| грузовик / грузовое авто / грузовой автомобиль | грузовик |
| спецтехника / спецтранспорт | спецтехника |
| мотоцикл / мототехника | мотоцикл |
| коммерческий транспорт | коммерческий транспорт |

То есть "авто" и "автомобиль" объединяем.

"Машина" держим отдельно.

## 6. Алиасы

Алиасы - это разные формулировки одного и того же интента.

Алиасы не становятся отдельными URL. Они покрываются внутри страницы: текстом, H2, FAQ, анкерами.

Пример:

```text
Интент: займ под ПТС
Технический адрес: /zaim-pod-pts/
```

Алиасы:

- займ под ПТС;
- займ под залог ПТС;
- займ по ПТС;
- ПТС займ;
- взять займ под ПТС;
- оформить займ под ПТС;
- получить займ под ПТС.

Пример:

```text
Интент: кредит под ПТС
Технический адрес: /kredit-pod-pts/
```

Алиасы:

- кредит под ПТС;
- кредит под залог ПТС;
- кредит по ПТС;
- ПТС кредит.

Пример:

```text
Интент: деньги под ПТС
Технический адрес: /dengi-pod-pts/
```

Алиасы:

- деньги под ПТС;
- деньги под залог ПТС;
- деньги по ПТС;
- ПТС деньги;
- получить деньги под ПТС.

## 7. Модификаторы

Модификаторы собираются один раз из всей очищенной семантики.

Не нужно собирать отдельный набор модификаторов под каждый интент.

Примеры модификаторов:

- онлайн;
- на карту;
- без визита;
- без справок;
- без подтверждения дохода;
- с плохой кредитной историей;
- с просрочками;
- без кредитной истории;
- без стоянки;
- без изъятия авто;
- авто остается у клиента;
- с правом пользования;
- срочно;
- рефинансирование;
- перезалог;
- для бизнеса;
- для юрлиц;
- для ИП;
- для физлиц;
- для самозанятых.

Для каждого модификатора задаем статус:

- отдельная страница;
- блок на странице;
- частый вопрос;
- алиас другого модификатора;
- рискованный;
- не использовать;
- проверить;
- отдельный интент.

Например:

| Модификатор | Решение |
| --- | --- |
| без справок | проверить / возможно отдельная страница |
| с плохой кредитной историей | проверить / возможно отдельная страница |
| без стоянки | важный модификатор, проверить как страницу |
| онлайн | часто достаточно блока |
| на карту | проверить |
| без отказа | рискованный, не использовать как страницу |
| рефинансирование | возможно отдельный интент |
| перезалог | возможно отдельный интент |

## 8. Правила применимости

Нельзя механически делать:

```text
каждый интент x каждый модификатор
```

Нужна таблица применимости.

Пример:

| Связка | Решение |
| --- | --- |
| займ под ПТС + без справок | проверить / возможно можно |
| кредит под ПТС + без справок | проверить |
| деньги под ПТС + на карту | проверить |
| займ под авто + без стоянки | проверить / возможно можно |
| заложить авто + на карту | скорее нет / проверить |
| любой интент + без отказа | не использовать |

Страницу создаем только если связка проходит фильтр:

- есть спрос;
- есть коммерческий смысл;
- можно честно продвигать;
- это не рискованный оффер;
- можно сделать не тонкий дубль.

## 9. URL-модель

Текущую структуру "город -> интент" не ломаем.

Используем:

```text
/{интент}/
/{интент}/{модификатор}/
/{город}/
/{город}/{интент}/
/{город}/{интент}/{модификатор}/
```

Пример:

```text
/zaim-pod-pts/
/zaim-pod-pts/bez-spravok/
/samara/
/samara/zaim-pod-pts/
/samara/zaim-pod-pts/bez-spravok/
```

## 10. Как обрабатывать очищенную семантику

Работаем сверху вниз.

1. Берем очищенную семантику и сортируем по частотности.
2. Берем самый сильный необработанный коммерческий запрос.
3. Определяем тип продукта.
4. Определяем объект залога.
5. Создаем коммерческий интент.
6. Собираем алиасы этого интента.
7. Отдельно выделяем модификаторы.
8. Отмечаем покрытые запросы как обработанные.
9. Переходим к следующему необработанному коммерческому запросу.
10. Повторяем, пока вся чистая коммерческая семантика не будет покрыта.

Пример:

```text
Запрос:
займ под залог ПТС

Разметка:
тип продукта: займ
объект залога: ПТС
интент: займ под ПТС
технический адрес: /zaim-pod-pts/
```

Алиасы:

- займ под ПТС;
- займ под залог ПТС;
- займ по ПТС;
- ПТС займ;
- взять займ под ПТС;
- оформить займ под ПТС.

Модификаторы:

- займ под ПТС без справок;
- займ под ПТС с плохой КИ;
- займ под ПТС без стоянки.

Интенты:

- интент;

- технический адрес;
- федеральный URL;
- статус;
- комментарий.

- алиас;
- интент.

- модификатор;
- алиасы;

- статус;
- комментарий.

Города:

- город;
- технический адрес;

Правила применимости:

- интент;
- модификатор;
- решение;

Правила генерации:

- правило;
- статус;
- комментарий.

Примеры правил:

| Правило | Статус |
| --- | --- |
| город x интент | можно |

| город x интент x рискованный модификатор | нельзя |
| федеральный хаб интента | обязателен |
| статья -> федеральный хаб | обязательная перелинковка |

## 13. Коротко

1. Все смысловые сущности в таблицах - на русском.
2. Технический адрес - отдельное поле.
3. Займ, кредит и деньги - разные коммерческие интенты.
4. ПТС, авто, машина, грузовик, спецтехника, мотоцикл - объекты залога.
5. Авто и автомобиль объединяем в объект "авто".
6. Машину держим отдельным объектом.
7. Алиасы не становятся URL.
8. Модификаторы собираем один раз из всей семантики.
9. Модификаторы применяются к интентам по правилам.
10. Федеральные хабы - каркас сайта.
11. Гео-страницы - основная масса programmatic SEO.
12. Структуру строим после покрытия очищенной семантики интентами, алиасами и модификаторами.
