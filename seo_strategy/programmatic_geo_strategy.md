# Стратегия построения SEO-структуры Bablocar

## 1. Структура сайта

На старте строим простую гео-сетку без федеральных хабов. Городской хаб - это сам URL второго уровня, например `/samara/`.

```text
Уровень 1
Главная
/
│
├── Уровень 2: /samara/
│   Городской хаб Самары
│   │
│   ├── Уровень 3: /samara/zaim-pod-pts/
│   │   Займ под ПТС в Самаре
│   │   │
│   │   └── Уровень 4: гео-модификаторы
│   │       ├── /samara/zaim-pod-pts/bez-spravok/
│   │       ├── /samara/zaim-pod-pts/s-plohoi-ki/
│   │       └── /samara/zaim-pod-pts/bez-stoyanki/
│   │
│   ├── Уровень 3: /samara/kredit-pod-pts/
│   │   Кредит под ПТС в Самаре
│   │
│   └── Уровень 3: /samara/dengi-pod-pts/
│       Деньги под ПТС в Самаре
│
├── Уровень 2: /kazan/
│   Городской хаб Казани
│   │
│   ├── Уровень 3: /kazan/zaim-pod-pts/
│   ├── Уровень 3: /kazan/kredit-pod-pts/
│   └── Уровень 3: /kazan/dengi-pod-pts/
│
└── Уровень 2: /moskva/
    Городской хаб Москвы
    │
    ├── Уровень 3: /moskva/zaim-pod-pts/
    ├── Уровень 3: /moskva/kredit-pod-pts/
    └── Уровень 3: /moskva/dengi-pod-pts/
```

## 2. Пояснение к структуре

### Городские хабы

Городской хаб - это верхняя страница города.

Пример:

```text
/samara/
```

SEO-роль:

- собирает все коммерческие интенты внутри одного города;
- помогает поисковику видеть локальный контур сайта;
- отделяет одну городскую сетку от другой.

Роль в структуре сайта:

- находится на втором уровне после главной;
- является родителем гео-страниц интентов в этом городе;
- связывает между собой направления внутри города.

На кого ссылается:

- на гео-страницы основных интентов в этом городе: `/samara/zaim-pod-pts/`, `/samara/kredit-pod-pts/`, `/samara/dengi-pod-pts/`.

Кто ссылается на него:

- главная: `/`;
- гео-страницы интентов этого города: `/samara/zaim-pod-pts/`, `/samara/kredit-pod-pts/`.

### Гео-страницы

Гео-страницы интентов - основная масса programmatic SEO на старте.

Пример:

```text
/samara/zaim-pod-pts/
```

SEO-роль:

- закрывает связку `город + коммерческий интент`;
- помогает собрать гео-коммерческую семантику;
- разделяет разные интенты внутри одного города;
- передает вес вниз на гео-модификаторы.

Роль в структуре сайта:

- находится внутри конкретного города;
- относится к конкретному коммерческому интенту;
- является дочерней страницей городского хаба;
- является родителем гео-модификаторов этого интента.

На кого ссылается:

- на родительский городской хаб: `/samara/`;
- на гео-модификаторы этого интента: `/samara/zaim-pod-pts/bez-spravok/`, `/samara/zaim-pod-pts/s-plohoi-ki/`;
- на соседние гео-страницы интентов в этом городе: `/samara/kredit-pod-pts/`, `/samara/dengi-pod-pts/`.

Кто ссылается на нее:

- родительский городской хаб: `/samara/`;
- гео-модификаторы этого интента: `/samara/zaim-pod-pts/bez-spravok/`, `/samara/zaim-pod-pts/s-plohoi-ki/`;
- соседние гео-страницы интентов в этом городе: `/samara/kredit-pod-pts/`, `/samara/dengi-pod-pts/`.

### Гео-модификаторы

Модификаторы применяются к интентам по правилам.

На старте используем гео-модификаторы:

```text
/samara/zaim-pod-pts/bez-spravok/
```

SEO-роль:

- закрывает связку `город + коммерческий интент + модификатор`;
- помогает покрывать уточненные сценарии внутри гео-сетки;
- разгружает основную гео-страницу интента от отдельных сценариев;
- усиливает глубину кластера в конкретном городе.

Роль в структуре сайта:

- находится внутри конкретного города;
- относится к конкретной гео-странице интента;
- является дочерней страницей связки `город + интент`;
- создается только если модификатор разрешен правилами применимости.

На кого ссылается:

- на родительскую гео-страницу интента: `/samara/zaim-pod-pts/`;
- на соседние гео-модификаторы этого интента: `/samara/zaim-pod-pts/s-plohoi-ki/`, `/samara/zaim-pod-pts/bez-stoyanki/`.

Кто ссылается на него:

- родительская гео-страница интента: `/samara/zaim-pod-pts/`;
- соседние гео-модификаторы этого интента: `/samara/zaim-pod-pts/s-plohoi-ki/`, `/samara/zaim-pod-pts/bez-stoyanki/`.

Отдельные федеральные страницы модификаторов пока не делаем. К федеральному слою можно вернуться позже.

## 3. Главный принцип

Мы строим структуру не под каждый отдельный ключ, а через систему:

```text
очищенная семантика
-> коммерческие интенты
-> алиасы интентов
-> модификаторы
-> правила применимости
-> городские хабы
-> гео-страницы интентов
-> гео-модификаторы
-> статьи / FAQ
```

Основная модель для programmatic SEO:

```text
города x коммерческие интенты x разрешенные модификаторы
```

Короткая математика матрицы:

```text
коммерческие интенты = продукты x объекты залога
продукты = займ / кредит / деньги
объекты залога = ПТС / авто / машина / грузовик / спецтехника / мотоцикл / коммерческий транспорт

3 продукта x 7 объектов = 21 базовый коммерческий интент
```

Модификаторы считаем не механически на все 21 интента, а только по применимости:

```text
город x базовый коммерческий интент x подходящие модификаторы
```

Формула для одного города:

```text
1 городской хаб
+ 21 интентная страница
+ сумма подходящих модификаторов по каждому из 21 интента
```

Оценка количества уникальных страниц под 1 город:

| Сценарий | Формула | Количество |
| --- | ---: | ---: |
| Минимальный | 1 хаб + 21 интент + 21 x 9 сильных модификаторов | 211 |
| Рабочий | 1 хаб + 21 интент + 21 x 15 подходящих модификаторов | 337 |
| Максимальный черновой | 1 хаб + 21 интент + 21 x 25 модификаторов | 547 |
| Теоретический потолок | 1 хаб + 21 интент + 21 x 33 модификатора | 715 |

Для планирования держим рабочий диапазон `300-350` страниц на 1 город, если брать не все подряд, а только подходящие модификаторы.

Для тестового запуска можно меньше:

```text
1 город = 1 хаб + 9 интентов + 9-12 модификаторов
примерно 90-120 страниц
```

## 4. Коммерческий интент

Коммерческий интент - это связка:

```text
тип продукта + объект залога
```

Типы продукта:

- займ;
- кредит;
- деньги.

`Автоломбард` и `заложить` пока не считаем отдельными типами продукта в pSEO-матрице. К ним можно вернуться позже как к отдельному слою или алиасам после проверки семантики.

Важное правило:

```text
займ != кредит != деньги
```

Это разные коммерческие интенты, а не алиасы друг друга.

Примеры:

- займ под ПТС;
- кредит под ПТС;
- деньги под ПТС;
- займ под авто;
- кредит под авто;
- деньги под авто;
- займ под машину;
- кредит под машину;
- деньги под машину.

## 5. Объекты залога

Объект залога - это то, под что пользователь хочет получить деньги.

Базовые объекты:

- ПТС;
- авто;
- машина;
- грузовик;
- спецтехника;
- мотоцикл;
- коммерческий транспорт.

Правила объединения:

| Формулировка | Объект |
| --- | --- |
| ПТС | ПТС |
| авто / автомобиль / автомобиля | авто |
| машина / машины | машина |
| машину | не объединяем с авто |
| грузовик / грузовое авто / грузовой автомобиль | грузовик |
| спецтехника / спецтранспорт | спецтехника |
| мотоцикл / мототехника | мотоцикл |
| коммерческий транспорт | коммерческий транспорт |

То есть "авто" и "автомобиль" объединяем.

"Машина" держим отдельно.

## 6. Алиасы

Алиасы - это разные формулировки одного и того же интента.

Алиасы не становятся отдельными URL. Они покрываются внутри страницы: текстом, H2, FAQ, анкерами.

Пример:

```text
Интент: займ под ПТС
Технический slug: zaim-pod-pts
```

Алиасы:

- займ под ПТС;
- займ под залог ПТС;
- займ по ПТС;
- ПТС займ;
- взять займ под ПТС;
- оформить займ под ПТС;
- получить займ под ПТС.

Пример:

```text
Интент: кредит под ПТС
Технический slug: kredit-pod-pts
```

Алиасы:

- кредит под ПТС;
- кредит под залог ПТС;
- кредит по ПТС;
- ПТС кредит.

Пример:

```text
Интент: деньги под ПТС
Технический slug: dengi-pod-pts
```

Алиасы:

- деньги под ПТС;
- деньги под залог ПТС;
- деньги по ПТС;
- ПТС деньги;
- получить деньги под ПТС.

## 7. Модификаторы

Модификаторы собираются один раз из всей очищенной семантики.

Не нужно собирать отдельный набор модификаторов под каждый интент.

Примеры модификаторов:

- онлайн;
- на карту;
- наличными;
- без визита;
- без справок;
- без подтверждения дохода;
- без прописки;
- без регистрации;
- без поручителей;
- без осмотра;
- по доверенности;
- с плохой кредитной историей;
- с просрочками;
- без кредитной истории;
- без стоянки;
- без изъятия авто;
- авто остается у клиента;
- с правом пользования;
- срочно;
- быстро;
- круглосуточно;
- без отказа;
- низкий процент;
- на длительный срок;
- рефинансирование;
- перезалог;
- для бизнеса;
- для юрлиц;
- для ООО;
- для ИП;
- для пенсионеров;
- для физлиц;
- для самозанятых.

Дополнительный источник: Яндекс Вебмастер, июль 2026. Подходящие новые формулировки добавляем в общий список модификаторов. Перед генерацией проверяем дубли, применимость к интентам и безопасную формулировку.

Для каждого модификатора задаем статус:

- отдельная гео-страница;
- блок на странице;
- частый вопрос;
- объединить с другим модификатором / учесть внутри страницы;
- рискованный;
- не использовать;
- проверить;
- отдельный интент.

## 8. Правила применимости

Нельзя механически делать:

```text
каждый интент x каждый модификатор
```

Нужна таблица применимости.

Пример:

| Связка | Решение |
| --- | --- |
| займ под ПТС + без справок | проверить / возможно можно |
| кредит под ПТС + без справок | проверить |
| деньги под ПТС + на карту | проверить |
| займ под авто + без стоянки | проверить / возможно можно |
| заложить авто + на карту | скорее нет / проверить |
| любой интент + без отказа | только с безопасной брокерской формулировкой |

Страницу создаем только если связка проходит фильтр:

- есть семантическое основание;
- есть коммерческий смысл;
- можно честно продвигать;
- рискованные формулировки не используются как обещание;
- можно сделать не тонкий дубль.

## 9. URL-модель

Текущую структуру "город -> интент" не ломаем.

Используем:

```text
/{город}/
/{город}/{интент}/
/{город}/{интент}/{модификатор}/
```

Пример:

```text
/samara/
/samara/zaim-pod-pts/
/samara/zaim-pod-pts/bez-spravok/
```

## 10. Как обрабатывать очищенную семантику

Работаем сверху вниз.

1. Берем очищенную семантику и сортируем по частотности.
2. Берем самый сильный необработанный коммерческий запрос.
3. Определяем тип продукта.
4. Определяем объект залога.
5. Создаем коммерческий интент.
6. Собираем алиасы этого интента.
7. Отдельно выделяем модификаторы.
8. Отмечаем покрытые запросы как обработанные.
9. Переходим к следующему необработанному коммерческому запросу.
10. Повторяем, пока вся чистая коммерческая семантика не будет покрыта.

Пример:

```text
Запрос:
займ под залог ПТС

Разметка:
тип продукта: займ
объект залога: ПТС
интент: займ под ПТС
технический адрес интента: zaim-pod-pts
планируемый URL: /samara/zaim-pod-pts/
```

Алиасы:

- займ под ПТС;
- займ под залог ПТС;
- займ по ПТС;
- ПТС займ;
- взять займ под ПТС;
- оформить займ под ПТС.

Модификаторы:

- займ под ПТС без справок;
- займ под ПТС с плохой КИ;
- займ под ПТС без стоянки;
- займ под ПТС срочно;
- займ под ПТС без отказа.
