# bablocar-seo

Главная база знаний SEO-проекта Bablocar.

README нужен как короткая карта решений: где мы находимся, по какой логике строим SEO, какие документы за что отвечают и что делать дальше. Подробная методология лежит в `seo_strategy/programmatic_geo_strategy.md`.

## Статус

Закрыты базовые этапы:

- продуктовая база;
- партнерские условия и безопасные формулировки;
- конкурентный блок;
- сбор и очистка семантики;
- базовая методология programmatic geo SEO.

Текущий рабочий этап: `SEO-стратегия`.

Ближайшая задача: разложить `semantic/clean_semantic.csv` на справочники и правила, затем собрать гео-матрицу страниц.

## Выжимка стратегии

Сначала структура, потом пояснения к уровням.

```text
Уровень 1
Главная
/
│
└── Уровень 2
    Федеральные хабы интентов
    │
    ├── /zaim-pod-pts/
    │   Займ под ПТС
    │   │
    │   ├── Федеральные модификаторы интента
    │   │   ├── /zaim-pod-pts/bez-spravok/
    │   │   ├── /zaim-pod-pts/s-plohoi-ki/
    │   │   └── /zaim-pod-pts/bez-stoyanki/
    │   │
    │   └── Уровень 3
    │       Гео-страницы этого интента
    │       │
    │       ├── /samara/zaim-pod-pts/
    │       │   Займ под ПТС в Самаре
    │       │   │
    │       │   └── Уровень 4
    │       │       Гео-модификаторы
    │       │       ├── /samara/zaim-pod-pts/bez-spravok/
    │       │       ├── /samara/zaim-pod-pts/s-plohoi-ki/
    │       │       └── /samara/zaim-pod-pts/bez-stoyanki/
    │       │
    │       ├── /kazan/zaim-pod-pts/
    │       │   Займ под ПТС в Казани
    │       │
    │       └── /moskva/zaim-pod-pts/
    │           Займ под ПТС в Москве
    │
    ├── /kredit-pod-pts/
    │   Кредит под ПТС
    │   │
    │   └── Гео-страницы этого интента
    │       ├── /samara/kredit-pod-pts/
    │       ├── /kazan/kredit-pod-pts/
    │       └── /moskva/kredit-pod-pts/
    │
    └── /dengi-pod-pts/
        Деньги под ПТС
        │
        └── Гео-страницы этого интента
            ├── /samara/dengi-pod-pts/
            ├── /kazan/dengi-pod-pts/
            └── /moskva/dengi-pod-pts/
```

Основная модель programmatic SEO:

```text
города x коммерческие интенты x разрешенные модификаторы
```

URL-модель:

```text
/{интент}/
/{интент}/{модификатор}/
/{город}/
/{город}/{интент}/
/{город}/{интент}/{модификатор}/
```

Рабочая логика:

```text
очищенная семантика
-> коммерческие интенты
-> алиасы интентов
-> модификаторы
-> правила применимости
-> федеральные хабы
-> гео-страницы
-> страницы с модификаторами
-> статьи / FAQ
```

Короткие правила из драфта стратегии:

- займ, кредит и деньги - разные коммерческие интенты;
- ПТС, авто, машина, грузовик, спецтехника, мотоцикл - объекты залога;
- авто и автомобиль объединяем в объект "авто";
- машину держим отдельным объектом;
- алиасы не становятся URL;
- модификаторы собираем один раз из всей семантики;
- модификаторы применяются к интентам по правилам;
- федеральные хабы - каркас сайта;
- федеральный хаб - это полноценный лендинг + хаб;
- гео-страницы - основная масса programmatic SEO.

Пример URL на интенте `займ под ПТС`:

```text
/zaim-pod-pts/
/zaim-pod-pts/bez-spravok/
/samara/
/samara/zaim-pod-pts/
/samara/zaim-pod-pts/bez-spravok/
```

## Роли документов

- `roadmap.md` - порядок этапов и текущий статус проекта.
- `product/product_brief.md` - продуктовые факты, ограничения и позиционирование.
- `partners/partner_conditions.md` - агрегированные условия партнеров.
- `partners/safe_claims.md` - безопасные и рискованные формулировки.
- `competitors/findings.md` - выводы по рынку и конкурентным подходам.
- `seo_strategy/programmatic_geo_strategy.md` - главный документ текущего этапа: стратегия, сущности, URL-модель, схема сайта, правила применимости и будущие справочники.

## Локальные данные

Семантика хранится локально и не коммитится:

- `semantic/raw_csv/` - сырые CSV-выгрузки;
- `semantic/combined_raw.csv` - объединенная сырая база;
- `semantic/clean_semantic.csv` - очищенная рабочая база;
- `semantic/strict_trash.csv` - удаленные нецелевые запросы;
