## Рабочая логика

Семантика не является готовой картой сайта. Она является словарем спроса, из которого мы выделяем управляемые сущности.

```text
очищенная семантика
-> справочник интентов
-> алиасы интентов
-> справочник модификаторов
-> справочник городов
-> правила применимости
-> правила генерации
-> гео-матрица pSEO
-> структура сайта
-> шаблоны страниц
-> ТЗ на страницы
```

Главная модель pSEO:

```text
город x коммерческий интент x разрешенный модификатор
```

Федеральные хабы нужны как каркас и смысловые родители. Основная SEO-масса строится через гео-страницы и гео-модификаторы.

## Схема на одном интенте

Пример показывает глобальную архитектуру на интенте `займ под залог ПТС`.

```text
/
└── /zaim-pod-pts/                         федеральный хаб интента
    ├── канонический интент: займ под залог ПТС
    ├── короткое название: займ под ПТС
    ├── алиасы на этой же странице:
    │   займ под ПТС, займ по ПТС, ПТС займ,
    │   взять / оформить / получить займ под ПТС
    │
    ├── гео-страницы этого интента
    │   ├── /moskva/zaim-pod-pts/          займ под залог ПТС в Москве
    │   │   ├── /moskva/zaim-pod-pts/bez-spravok/   без справок
    │   │   ├── /moskva/zaim-pod-pts/s-plohoi-ki/    с плохой кредитной историей
    │   │   ├── /moskva/zaim-pod-pts/bez-stoyanki/   без стоянки
    │   │   └── /moskva/zaim-pod-pts/na-kartu/       на карту
    │   │
    │   ├── /samara/zaim-pod-pts/          займ под залог ПТС в Самаре
    │   └── /kazan/zaim-pod-pts/           займ под залог ПТС в Казани
    │
    ├── поперечная навигация по городу
    │   └── /moskva/ -> /moskva/kredit-pod-pts/,
    │                 /moskva/dengi-pod-pts/,
    │                 /moskva/zaim-pod-avto/
    │
    ├── соседние федеральные интенты
    │   ├── /kredit-pod-pts/
    │   ├── /dengi-pod-pts/
    │   └── /zaim-pod-avto/
    │
    └── поддерживающий слой
        ├── статьи / FAQ / инструкции по ПТС
        └── калькуляторы / околоавто-инструменты
```

Короткие правила:

- алиас не создает отдельный URL;
- город создает дочернюю гео-страницу интента;
- модификатор может создать дочернюю страницу от `город + интент`, если он разрешен правилами применимости;
- городской хаб, статьи, FAQ и инструменты усиливают коммерческие страницы ссылками, но не ломают основное дерево;
- финальная структура строится из справочников и правил, а не из отдельных ключевых фраз.

## Роли документов

- `roadmap.md` - порядок этапов и текущий статус проекта.
- `product/product_brief.md` - продуктовые факты, ограничения и позиционирование.
- `partners/partner_conditions.md` - агрегированные условия партнеров.
- `partners/safe_claims.md` - безопасные и рискованные формулировки.
- `competitors/findings.md` - выводы по рынку и конкурентным подходам.
- `seo_strategy/programmatic_geo_strategy.md` - главный документ текущего этапа: стратегия, сущности, URL-модель, схема сайта, правила применимости и будущие справочники.

## Локальные данные

Семантика хранится локально и не коммитится:

- `semantic/raw_csv/` - сырые CSV-выгрузки;
- `semantic/combined_raw.csv` - объединенная сырая база;
- `semantic/clean_semantic.csv` - очищенная рабочая база;
- `semantic/strict_trash.csv` - удаленные нецелевые запросы;
- `semantic/semantic_review.xlsx` - таблица для просмотра clean-базы.
