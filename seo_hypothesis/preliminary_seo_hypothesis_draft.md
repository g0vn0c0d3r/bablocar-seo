# Предварительная SEO-гипотеза Bablocar

Статус: черновик для обсуждения.

Документ фиксирует рабочую гипотезу после продуктового брифа, безопасных формулировок, партнерских условий и первичного конкурентного анализа. Это не финальная SEO-стратегия, не карта сайта и не техническое задание на страницы.

## Главная ставка

Bablocar должен строиться не как один лендинг "займ под ПТС", а как SEO-система вокруг заявки на подбор займа под залог ПТС / автомобиля.

Основной механизм роста:

```text
продуктовые кластеры
x география
x безопасные коммерческие интенты
x полезные инструменты
```

Всю матрицу нельзя публиковать автоматически. Сначала нужна обработка семантики, кластеризация, скоринг страниц-кандидатов и только потом публикация волнами.

## 1. Базовые кластеры

Первый слой страниц:

| Кластер | Роль |
| --- | --- |
| `займ под залог ПТС` | Главный коммерческий кластер. |
| `займ под ПТС` / `ПТС займ` | Вероятно алиасы главного кластера, не обязательно отдельные страницы. |
| `деньги под залог ПТС` | Отдельный коммерческий интент, может требовать отдельной страницы. |
| `кредит под залог ПТС` | SEO-интент, но с аккуратным объяснением, что продукт может оформляться как заем у партнера. |
| `займ под залог авто` | Широкий кластер вокруг автомобиля как залога. |
| `автоломбард / займ в автоломбарде` | Осторожный кластер: Bablocar не классический автоломбард. |
| `рефинансирование под залог авто` | Отдельный сценарий для действующего займа / кредита. |
| `для бизнеса / ИП / юрлиц` | Отдельный слой, особенно из-за лимита до 5 млн для бизнеса. |

Не стоит автоматически плодить одновременно `pts-zaim`, `zaim-pod-pts`, `zaim-pod-zalog-pts`, если они закрывают один и тот же SERP. Это риск каннибализации.

## 2. Гео-сетка

Сильная гео-сетка подходит Bablocar, потому что продукт работает по РФ и конкуренты активно используют гео-структуры.

Базовая логика:

```text
/город/категория/
/город/категория/интент/
```

Примеры:

```text
/samara/zaym-pod-zalog-pts/
/samara/zaym-pod-zalog-pts/na-kartu/
/samara/zaym-pod-zalog-pts/bez-spravok/
/samara/zaym-pod-zalog-pts/s-plohoy-kreditnoy-istoriey/
/samara/zaym-pod-zalog-pts/bez-stoyanki/
/samara/zaym-pod-zalog-pts/bez-izyatiya-avto/
```

Условие: каждая страница должна отличаться не только городом и H1.

Городская страница должна иметь хотя бы часть уникального смысла:

- доступность онлайн-заявки в городе;
- региональное уточнение без выдуманных офисов;
- популярные сценарии в городе;
- локальные FAQ;
- примеры расчетов;
- перелинковку на соседние интенты;
- корректный юридический блок;
- CTA рядом с калькулятором.

Если город ничего не меняет и страница отличается только топонимом, страница рискует стать doorway.

## 3. Теги / интенты

Теги делим на группы по уровню риска.

### Зеленые

| Тег | Почему можно |
| --- | --- |
| `онлайн` | Совпадает с продуктом и воронкой Bablocar. |
| `на карту` | Можно использовать, но без обещания мгновенного перевода. |
| `без визита` | Сильный продуктовый тезис про онлайн-онбординг. |
| `без справок` | Можно с уточнением про документы на авто и условия партнера. |
| `авто остается у клиента` | Важный оффер, но без абсолютной гарантии всех партнеров. |
| `без стоянки` | Нормальный SEO-тег, если формулировать как подбор предложений с сохранением пользования авто. |
| `без изъятия` | Нормальный SEO-тег, если не обещать это как безусловную гарантию. |
| `для бизнеса` | Подтверждено продуктовой логикой и лимитом до 5 млн для бизнеса. |
| `рефинансирование` | Отдельный реальный сценарий. |

### Желтые

| Тег | Как использовать |
| --- | --- |
| `плохая кредитная история` | "Можно оставить заявку", не "одобрим". |
| `просрочки` | Только индивидуальное рассмотрение. |
| `срочно` | Лучше "быстрое решение", не "деньги за 30 минут". |
| `без отказа` | Можно только как SEO-интент, но заголовки и текст должны быть безопасными. |

### Красные / почти стоп

| Тег | Почему опасно |
| --- | --- |
| `100% одобрение` | Нельзя обещать. |
| `деньги за 15/30 минут` | Рискованно из-за проверок, решения партнера и периода охлаждения. |
| `без документов` | Противоречит продукту: документы на транспорт нужны. |
| `без проверки` | YMYL-риск и конфликт с реальным процессом. |
| `на чужое авто без собственника` | Фрод / юридический риск. |

## 4. Как страницы должны отличаться

Нельзя строить страницы по принципу "один шаблон + замена города".

Примеры различий:

| Страница | Уникальный фокус |
| --- | --- |
| `.../na-kartu/` | Способ получения, карта, когда возможен перевод, что зависит от партнера. |
| `.../bez-vizita/` | Онлайн-заявка, фото авто, документы, AI-распознавание. |
| `.../bez-spravok/` | Какие справки могут не понадобиться, какие документы все равно нужны. |
| `.../bez-stoyanki/` | Подбор предложений, где клиент продолжает пользоваться автомобилем; условия партнера. |
| `.../bez-izyatiya-avto/` | Сохранение пользования автомобилем без жесткого обещания "никогда не забирают". |
| `.../s-plohoy-ki/` | Почему залог может помочь, но решение не гарантируется. |
| `.../refinansirovanie/` | Действующий займ / кредит, переоформление, не второй независимый залог. |
| `.../dlya-biznesa/` | Лимит до 5 млн, ИП / ООО, коммерческий транспорт. |

Если для тега нельзя написать отдельную полезную логику, страницу лучше не индексировать.

## 5. Публикационная стратегия

Не нужно заливать сразу тысячи страниц.

Волны:

| Волна | Что публикуем |
| --- | --- |
| 1 | Федеральные ядровые страницы: ПТС, авто, деньги, кредит, онлайн. |
| 2 | Топ-10 городов x основной кластер. |
| 3 | Топ-10 городов x безопасные теги. |
| 4 | Топ-30/50 городов x только подтвержденные сильные интенты. |
| 5 | Сложные интенты: плохая КИ, просрочки, без отказа - после ручной проверки шаблонов. |
| 6 | Калькуляторы и авто-инструменты. |

Смысл не в магическом лимите страниц, а в контроле качества. Массовые страницы без добавленной пользы могут выглядеть как doorway / scaled content.

## 6. Генератор

Идея генератора рабочая, но ему нужен жесткий контур безопасности.

Обязательные справочники:

- канонические категории;
- алиасы запросов;
- теги и уровень риска;
- города и регионы;
- безопасные формулировки;
- ставки / суммы / сроки только из проверенного источника;
- стоп-фразы;
- правила CTA;
- правила юридических блоков.

QC должен ловить:

- выдуманные ставки;
- "одобрение 98%";
- "без отказа" как обещание;
- фейковые отзывы;
- отсутствие периода ставки;
- отсутствие юридического раскрытия;
- слишком похожие страницы;
- дубли H1 / title;
- городские страницы без локальной пользы;
- конфликт между калькулятором, текстом и безопасными формулировками.

Сгенерированные отзывы нельзя публиковать как реальные отзывы. Максимум - "пример ситуации клиента", и только если формат не вводит пользователя в заблуждение.

## 7. Калькуляторы и авто-инструменты

Калькуляторы - хороший второй контур.

Сначала:

```text
/calculators/zaym-pod-zalog-pts/
/calculators/zaym-pod-zalog-avto/
```

Рядом с расчетом обязательно нужен CTA:

```text
Рассчитать -> оставить заявку -> загрузить фото авто / документы
```

Позже можно думать про околоавто:

- проверка авто;
- проверка залога;
- проверка VIN;
- ОСАГО / КАСКО;
- расчет стоимости владения;
- штрафы / ограничения, если есть легальный источник данных.

Этот слой не должен отвлекать от главной коммерческой сетки. Это расширение, когда ядро уже собрано.

## 8. Следующий рабочий шаг

Дальше нужно:

1. Нормализовать всю семантику.
2. Склеить CSV в один массив.
3. Удалить мусор: игры, недвижимость, стиральные машины, банковский общак.
4. Дедуплицировать запросы.
5. Разметить каждый запрос: `кластер / интент / гео / риск / коммерческий или информационный / нужен ли лендинг`.
6. Построить не сайтмап, а матрицу возможностей: `страница-кандидат -> спрос -> риск -> уникальность -> приоритет волны`.

Главная мысль: geo pSEO подходит Bablocar, но только через управляемую матрицу, а не через генерацию всех комбинаций.
