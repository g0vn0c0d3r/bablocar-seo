# Правила генерации страниц

Файл фиксирует правила масштабирования programmatic SEO-страниц: как создаются города, интенты и модификаторы.

## Логика создания и масштабирования страниц

Город - верхний уровень масштабирования.

Сайт растет слоями:

```text
город
->
интент
->
модификатор
```

Создали город - появилась городская страница:

```text
/samara/
```

Создали интент - он появился во всех созданных городах:

```text
/samara/zaim-pod-pts/
/kazan/zaim-pod-pts/
/moskva/zaim-pod-pts/
```

Создали модификатор - он появился внутри нужного интента во всех созданных городах:

```text
/samara/zaim-pod-pts/s-plohoi-ki/
/kazan/zaim-pod-pts/s-plohoi-ki/
/moskva/zaim-pod-pts/s-plohoi-ki/
```

Если потом добавить новый город, он получает уже созданные интенты и модификаторы:

```text
/ufa/
/ufa/zaim-pod-pts/
/ufa/zaim-pod-pts/s-plohoi-ki/
```

Порядок создания не важен. Новая сущность достраивает недостающие страницы по уже согласованной логике.

**Городские хабы**

Меняются: город, URL, Title, H1, ссылки на направления внутри города, карточки заявок, отзывы, FAQ и SEO-блок.
Не меняются: порядок блоков, сквозные блоки, логика Hero, CTA, документов, преимуществ и процесса заявки.

**Гео-интенты**

Меняются: город, URL, Title, H1, ссылки на модификаторы этого интента, FAQ и SEO-блок.
Не меняются: структура страницы, сквозные блоки и базовая логика интента.

**Гео-модификаторы**

Меняются: город, интент, модификатор, URL, Title, H1, связанные ссылки, FAQ и SEO-блок.
Не меняются: порядок блоков, сквозные блоки, общая логика карточек, отзывов, документов и CTA.

Важно: генерация не должна создавать разные по смыслу страницы одного типа. Если добавляется новый город, меняются только городские переменные и связанные с ними блоки. Структура, порядок блоков и логика текста остаются одинаковыми.
