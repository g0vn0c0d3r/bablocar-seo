# Live publication smoke check

Короткая проверка после публикации новых pSEO-страниц.

## Когда запускать

- после добавления нового интента;
- после добавления нового города;
- после добавления нового модификатора;
- после исправления sitemap или перелинковки.

## Что проверяем

1. Sitemap:
   - `https://bablocar.ru/sitemap.xml` отдает sitemap index;
   - `https://bablocar.ru/sitemap-0.xml` отдает 200;
   - URL из sitemap считываются без дублей;
   - количество URL совпадает с ожидаемой сеткой.

2. Live-страницы из sitemap:
   - каждая страница отдает 200;
   - нет `noindex`;
   - есть `title`;
   - есть `description`;
   - есть `H1`;
   - canonical есть и совпадает с URL из sitemap.

3. Связка с хабами:
   - городской хаб содержит ссылки на базовые интенты города;
   - базовая интентная страница содержит ссылки на свои модификаторы;
   - в городском хабе есть блок `Основные направления в {город}`.

## Что не входит

- не проверяем юридическую чистоту формулировок;
- не оцениваем ставки, сроки, обещания и офферы;
- не сравниваем тексты на шаблонность;
- не делаем визуальный аудит и скриншоты;
- не скачиваем live-страницы глубже, чем нужно для базовых SEO-сигналов.

## Формат результата

Коротко:

```text
Sitemap: {N} URL, дублей нет.
Live верхнего слоя: {N}/{N} отдают 200.
SEO: title/description/H1/canonical/noindex - без ошибок.
Связка: хаб -> базовые интенты - без ошибок.
```

Если есть ошибки, перечислять только проблемные URL и тип проблемы.
