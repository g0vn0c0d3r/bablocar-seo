# Аудит коммерческих лендингов

Файл собирает короткие аудиты готовых страниц без скриншотов.

Проверяем только структуру: наличие блоков, порядок, количество элементов, лимиты карточек и видимость блоков. Контентные настройки, которые можно поменять в JSON или админке, здесь не считаем проблемой: метаданные, H1/H2, подводки, SEO-текст и формулировки внутри FAQ.

Общее правило для карточек: внутри одного блока карточки не должны быть одинаковыми. Это касается отзывов, примеров заявок, соседних направлений и модификаторов.

Если в админке создано больше сущностей, чем выводится на странице, блок должен показывать ограниченный набор карточек. Текущая тестовая матрица: 1 город, 8 интентов и 8 модификаторов. Для городского хаба это проверяет, что блок направлений выводит ограниченный набор карточек, а не все созданные интенты.

Городской хаб, гео-интенты и гео-модификаторы ниже актуализированы после пересборки тестовой матрицы.

В таблицах: `+` - блок ок, `-` - есть проблема.

- `https://bblcr-seo-public.outdxb.com/ekb/kredit-pod-zalog-mashiny`

Тип страницы: гео-интент.

Проверка из HTML страниц:

- все 8 страниц отдают нужные структурные блоки;
- блок `Примеры заявок / обращений`: 4 карточки на каждой странице, точных дублей внутри страниц и между интентами не найдено;
- блок `Отзывы клиентов`: 8 карточек на каждой странице, точных дублей внутри страниц и между интентами не найдено;
- блок `Связанные сценарии / модификаторы`: выводится 6 ссылок из 8 созданных модификаторов;
- блок `FAQ`: 5-6 вопросов на странице.

Проверка пересечения страниц:

Методика: из HTML извлечен видимый текст страницы, включая текстовые `children` из Next flight-данных; `script`, `style`, `svg`, шапка и футер исключены. Текст нормализован до слов, дальше сравнивались 5-словные шинглы. Основная метрика для оценки похожести - пересечение меньшей страницы, дополнительная метрика - Jaccard.

Рабочая шкала для коммерческих лендингов:

| Пересечение меньшей страницы | Интерпретация |
|---:|---|
| до 30% | низкое пересечение |
| 30-50% | умеренная шаблонность |
| 50-70% | нормальная коммерческая шаблонность, если различаются карточки, FAQ и сценарные блоки |
| 70-80% | зона внимания: проверить, не отличается ли страница только ключом |
| 80%+ | высокий риск дубля |
| 90%+ | почти полный дубль / doorway-риск |

Результат по 8 интентам: все 28 пар находятся в рабочем диапазоне коммерческой шаблонности. Пересечение меньшей страницы по 5-словным шинглам: 58,3-69,0%; Jaccard: 40,6-52,4%. Для тестовой матрицы это нормальный результат: общий лендинговый каркас совпадает, но карточки заявок и отзывы разведены, FAQ держится на уровне 5-6 вопросов.

## Гео-модификаторы Екатеринбурга

Статус: актуальный аудит после пересборки тестовой матрицы.

Проверены 64 страницы: 8 интентов × 8 модификаторов. Все страницы отдают `200`.

Тип страницы: гео-модификатор.

Проверка из HTML и визуального рендера:

- все структурные блоки на месте;
- `Предложения партнеров`: 3 предложения на странице, нужные колонки есть;
- `Примеры заявок / обращений`: 4 карточки, точных дублей внутри страницы нет;
- `Отзывы клиентов`: 8 карточек, точных дублей внутри страницы нет;
- `Связанные сценарии`: 6 ссылок;
- `FAQ`: 5-6 вопросов, дублей внутри страницы нет;
- визуально выборочно проверены 4 страницы: порядок блоков и финальный CTA соответствуют текущему дизайну.

Между разными модификаторами есть переиспользование карточек и отзывов. По текущему правилу это не считается ошибкой, потому что правило про дубли относится к карточкам внутри одного блока на странице. Если захотим сделать сетку строже, можно отдельно ротировать заявки и отзывы между модификаторами одного интента.

Фактический визуальный порядок: Hero с калькулятором -> как получить деньги -> преимущества -> партнеры -> заявки -> документы -> транспорт -> отзывы -> связанные сценарии -> SEO-блок -> FAQ -> финальный CTA.

Проверка пересечения страниц:

Методика такая же, как для гео-интентов: видимый текст + текстовые `children` из Next flight-данных, затем 5-словные шинглы. Основная метрика - пересечение меньшей страницы.

Результат по 64 модификаторам: пересечение меньшей страницы 62,2-75,0%; Jaccard 44,8-59,1%. Это рабочая коммерческая шаблонность с частью пар в зоне внимания 70-75%, но без высокого риска: пар выше 80% нет.
