# План конкурентной аналитики Bablocar

## Цель

Понять, как конкуренты в нише займов под залог ПТС / авто собирают SEO-трафик, какие структуры используют, какие интенты закрывают и какие подходы можно потом проверить для Bablocar.

На этом этапе не строим финальную SEO-стратегию и не делаем карту сайта Bablocar. Сначала собираем факты по конкурентам.

## 2. Технический срез: robots / sitemap / URL

Что делаем:

- проверяем, открывается ли сайт;
- проверяем, есть ли robots.txt;
- проверяем, есть ли sitemap.xml;
- проверяем, есть ли sitemap index;
- считаем, сколько URL найдено в sitemap;

Что важно:

- количество URL не придумываем на глаз.

## 6. Пересечение и уникальность ПТС-страниц

Что делаем:

Для конкурентов с несколькими ПТС-страницами считаем пересечение контента:

- `title`, `description`, H1;
- H2/H3-структура;
- основной видимый текст;
- повторяющиеся абзацы / строки;
- 5-словные шинглы;
- FAQ-вопросы;
- блоки документов, требований и условий;
- CTA, формы и стандартные доверительные блоки.

Что важно:

- повтор документов, требований, дисклеймеров и юридических блоков может быть нормальным;
- риск возникает, когда страницы под разные интенты отличаются только H1 и заменой слов `займ`, `кредит`, `деньги`;
- высокое пересечение нужно подтверждать цифрами, а не оценивать на глаз.

Ориентировочная шкала:

| Пересечение | Интерпретация |
| --- | --- |
| До 30% | Страницы достаточно разные |
| 30-50% | Умеренная шаблонность |
| 50-70% | Высокий риск каннибализации |
| 70%+ | Почти дубль / doorway-риск |

Что ожидаем на выходе:

- какие идеи стоит проверить семантикой.

В нем:

- вопросы к продукту;
- сайты для повторного глубокого разбора.

## Порядок работы

Анализ делаем не одной большой задачей, а по шагам:

1. Список конкурентов.
2. robots / sitemap / URL-срез.
3. Структурный SEO-анализ.
4. Анализ ключевых страниц.
5. Контентная архитектура.
6. Пересечение и уникальность ПТС-страниц.
7. Поисковые риски и качество.
8. Сводные выводы.

На старте не расставляем приоритеты. Смотрим всех конкурентов по единой рамке. Приоритеты появятся после первого фактического среза.
