# План конкурентной аналитики Bablocar

## Цель

Понять, как конкуренты в нише займов под залог ПТС / авто собирают SEO-трафик, какие структуры используют, какие интенты закрывают и какие подходы можно потом проверить для Bablocar.

На этом этапе не строим финальную SEO-стратегию и не делаем карту сайта Bablocar. Сначала собираем факты по конкурентам.

## 2. Технический срез: robots / sitemap / URL

Что делаем:

- проверяем, открывается ли сайт;
- проверяем, есть ли robots.txt;
- проверяем, есть ли sitemap.xml;
- проверяем, есть ли sitemap index;
- считаем, сколько URL найдено в sitemap;
- фиксируем, какие URL-паттерны видны.

Что важно:

- если sitemap не найден или недоступен, так и пишем;
- количество URL не придумываем на глаз.

## 4. Анализ ключевых страниц

Что делаем:

Для каждого конкурента выбираем несколько ключевых страниц и разбираем:

- какой интент закрывает страница;
- тип страницы: лендинг, каталог, карточка, статья, гео, FAQ;
- как компания описывает свою роль;
- какие УТП использует;
- есть ли форма заявки;
- есть ли калькулятор;
- есть ли FAQ;
- есть ли блок документов;
- есть ли блок требований;
- есть ли отзывы, лицензии, рейтинги, партнеры;
- как пишут про авто у клиента;
- как пишут про сумму;
- как пишут про срок;
- как пишут про ставку;
- как пишут про онлайн-заявку;
- какие рискованные обещания используют.

## 5. Сводные выводы

Что делаем:

После просмотра всех сайтов собираем общий вывод:

- какие типы игроков доминируют;
- какие структуры чаще встречаются;
- кто использует гео;
- кто использует programmatic SEO;
- какие интенты чаще всего покрываются;
- какие блоки повторяются на сильных страницах;
- какие офферы встречаются чаще всего;
- какие рискованные обещания встречаются;
- какие идеи стоит проверить семантикой.

Что ожидаем на выходе:

Документ `findings.md`.

В нем:

- лучшие идеи для Bablocar;
- рискованные приемы;
- типы страниц для проверки семантикой;
- вопросы к продукту;
- сайты для повторного глубокого разбора.

## Порядок работы

Анализ делаем не одной большой задачей, а по шагам:

1. Список конкурентов.
2. robots / sitemap / URL-срез.
3. Структурный SEO-анализ.
4. Анализ ключевых страниц.
5. Сводные выводы.

На старте не расставляем приоритеты. Смотрим всех конкурентов по единой рамке. Приоритеты появятся после первого фактического среза.
